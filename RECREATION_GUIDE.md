# Recreation Guide for CAR (Claude Artifact Repository)

<!-- cspell:words cmdk exceljs katex papaparse plotly turbopack -->

This guide recreates this repository from scratch with the current toolchain:
Next.js 16, React 19, Tailwind CSS 4, shadcn, pnpm, TypeScript, Prettier, Husky,
and the repo's canonical ESLint policy.

Version facts checked on 2026-06-16:

- `next`: 16.2.9
- `react` / `react-dom`: 19.2.7
- `tailwindcss` / `@tailwindcss/postcss`: 4.3.1
- `zod`: 4.4.3
- `@hookform/resolvers`: 5.4.0
- `typescript`: 6.0.3
- `typescript-eslint`: 8.61.1
- `eslint-formatter-unix`: 9.0.1
- `eslint`: latest is 10.5.0, but this repo uses ESLint 9 because ESLint 10 is not compatible with the current `@typescript-eslint` setup.

References:

- [Next.js installation](https://nextjs.org/docs/app/getting-started/installation)
- [Tailwind CSS with PostCSS](https://tailwindcss.com/docs/installation/using-postcss)

## Prerequisites

- `nvm`
- `git`
- Corepack-managed `pnpm`

## Step 1: Node.js Environment Setup

Install and use the latest LTS version of Node.js:

```bash
nvm install --lts
nvm use --lts
```

Pin the selected Node.js version:

```bash
node -v > .nvmrc
```

Enable Corepack and prepare pnpm:

```bash
corepack enable pnpm
pnpm --version
```

Ensure pnpm is the package manager signal for tools that auto-detect from lock
files:

```bash
rm -f bun.lock package-lock.json yarn.lock
```

Do not run `corepack use pnpm@latest` from inside this checkout if Corepack
detects a parent workspace outside the repository. In that case it may try to
write a `package.json` outside this repo. Use the active Corepack-managed
`pnpm`, or set the package manager field from the actual repo root only.

## Step 2: Remove Generated shadcn Files

Before running shadcn, remove previously generated shadcn files so the registry
starts from a clean state instead of prompting for overwrites or merging stale
component output:

```bash
rm -rf components.json src/components/ui src/hooks/use-mobile.ts
```

Do not remove app-owned files such as `src/components/artifact.tsx`.

## Step 3: Initialize Project via shadcn

Use the current shadcn Next template. This bootstraps Next.js 16, React 19,
TypeScript, Tailwind CSS 4, App Router, and the usual `src`, `app`, `components`,
and `lib` paths. The files removed in the previous step should be regenerated
from a clean state.

```bash
pnpm dlx shadcn@latest init -t next --defaults
```

Do not select old repository-specific values when shadcn changes its prompts.
Accept the current shadcn defaults instead. The generated `components.json`
should be treated as the source of truth for component library, preset, base
color, menu settings, and future shadcn default fields.

If shadcn creates a nested `car` directory, move the generated files to the repo
root:

```bash
mv car/* car/.* . 2>/dev/null || true
rmdir car
```

## Step 5: Install Dependencies

Use latest stable dependencies unless this guide explicitly pins otherwise.

Install app dependencies:

```bash
pnpm add zod@latest @hookform/resolvers@latest \
  @base-ui/react shadcn@latest ag-grid-community ag-grid-react \
  babel-plugin-react-compiler chart.js class-variance-authority clsx cmdk cspell d3 date-fns \
  embla-carousel-react exceljs framer-motion input-otp katex lodash lucide-react mammoth mathjs \
  next-themes papaparse plotly.js react-day-picker react-hook-form react-resizable-panels \
  recharts sonner tailwind-merge tensorflow three tone vaul xlsx
```

Install dev dependencies:

```bash
pnpm add -D @types/node@latest @types/react@latest @types/react-dom@latest \
  @tailwindcss/postcss@latest tailwindcss@latest tw-animate-css@latest \
  @tsconfig/next@latest @tsconfig/strictest@latest typescript@latest \
  prettier@latest prettier-plugin-tailwindcss@latest pretty-quick@latest husky@latest
```

Install ESLint dependencies. Keep ESLint on v9; do not use ESLint 10 yet.

```bash
pnpm add -D eslint@^9 @eslint/js@^9 eslint-formatter-unix@latest \
  @eslint/eslintrc eslint-config-next@latest eslint-config-prettier@latest \
  typescript-eslint@latest @tanstack/eslint-plugin-query@latest \
  eslint-import-resolver-typescript@latest eslint-plugin-import-x@latest \
  eslint-plugin-promise@latest eslint-plugin-react@latest eslint-plugin-react-hooks@latest \
  eslint-plugin-react-refresh@latest eslint-plugin-simple-import-sort@latest \
  eslint-plugin-unused-imports@latest globals@latest
```

If pnpm reports `ERR_PNPM_UNEXPECTED_STORE`, reinstall dependencies with the
active pnpm before adding more packages:

```bash
pnpm install
```

If pnpm asks to remove and recreate `node_modules`, accept it. This only links
ignored dependencies again and avoids mixing stores from different pnpm versions.
If the error persists because a tool wants the project-local store, remove
`node_modules` and reinstall with:

```bash
pnpm install --store-dir .pnpm-store
```

## Step 6: Configure Husky and Git Hooks

Initialize Husky:

```bash
pnpm exec husky init
```

Set the pre-commit hook:

```bash
echo "pnpm exec pretty-quick --staged" > .husky/pre-commit
```

## Step 7: Install All shadcn Components

Regenerate all registry components so `src/components/ui` matches this repo.
The earlier cleanup step should mean there are no stale generated files to
merge. Keep `--overwrite` as a harmless fallback if shadcn still detects an
existing generated file:

```bash
pnpm dlx shadcn@latest add --all -y --overwrite
```

If the current shadcn registry output differs from older repository
`src/components/ui` files, prefer the newly generated shadcn output. Do not
hand-fix generated component lint failures during recreation; those failures
usually mean the upstream registry has drifted from this repo's stricter ESLint
policy.

## Step 8: Configure Project Scripts

Update `package.json` scripts. Preserve the Husky `prepare` script and include
the unix formatter in the lint script.

```json
{

    "build": "next build",
    "dev": "next dev --turbopack",
    "format": "prettier --write --ignore-unknown .",
    "lint": "eslint --cache --fix --format unix .",
    "prepare": "husky; git config blame.ignoreRevsFile .git-blame-ignore-revs",
    "spellcheck": "cspell --config=cspell.json \"**/*.{cjs,js,md,mjs,ts,tsx}\" --no-progress --show-context --show-suggestions",
    "start": "next start",
    "typecheck": "tsc --noEmit"
  }
}
```

## Step 9: Update App Code

`shadcn init` generates `src/app/layout.tsx`, `src/app/globals.css`, and
`src/lib/utils.ts`. Verify `globals.css` starts with the Tailwind CSS 4 imports
and the current shadcn default CSS import:

```css
@import "tailwindcss";
@import "tw-animate-css";
@import "shadcn/tailwind.css";
```

Replace `src/app/page.tsx` with:

```tsx
"use client";

import dynamic from "next/dynamic";

const Artifact = dynamic(() => import("@/components/artifact"), { ssr: false });

export default function Home() {
  return (
    <main className="min-h-screen antialiased">
      <Artifact />
    </main>
  );
}
```

Create `src/components/artifact.tsx`:

```tsx
function App() {
  return (
    <div className="mx-auto grid max-w-lg gap-4 px-4 py-8">

export default App;
```

## Step 11: Configure ESLint

Do not use the basic generated ESLint config. After installing the ESLint
dependencies, copy this repo's `eslint.config.mjs` verbatim.

Reason: the config encodes the repo's canonical policy, including Next.js 16 flat
config presets, type-aware TypeScript rules, import-x resolver policy, React
rules, React Hooks rules, promise rules, TanStack Query rules, unused-import
deduping, scoped exceptions, and validation contracts. A shortened config will
not recreate the repo's behavior.

## Step 12: Verify Installation

Remove ignored build output before verification so cspell does not scan generated
`out/` or `.next/` JavaScript:

```bash
rm -rf out .next
```

Run:

```bash
pnpm run format
pnpm run spellcheck
pnpm run typecheck
pnpm run build
```

`pnpm run lint` is useful as a policy check, but it is not a required recreation
gate after refreshing shadcn components. If it reports issues only in generated
`src/components/ui` files, keep the newly generated registry output instead of
hand-editing it back to older repository component files.

Then start the dev server:

```bash
pnpm run dev
```

Open <http://localhost:3000>. The page should render the "Hello World" artifact
placeholder.

## Common Issues

### ESLint `unix` formatter missing

ESLint 9 no longer ships the unix formatter in core. Install
`eslint-formatter-unix` and keep the lint script as:

```json
"lint": "eslint --cache --fix --format unix ."
```

### ESLint 10 selected by `@latest`

Do not use `eslint@latest` until the `@typescript-eslint` stack supports it for
this repo. Use:

```bash
pnpm add -D eslint@^9
```

### shadcn creates a nested directory

Move files from the nested `car` directory into the repository root:

```bash
mv car/* car/.* . 2>/dev/null || true
rmdir car
```

### pnpm not found

Enable Corepack:

```bash
corepack enable pnpm
```

### pnpm store mismatch

If pnpm reports `ERR_PNPM_UNEXPECTED_STORE`, run:

```bash
pnpm install
```

Accept the prompt to recreate `node_modules`. Do not hand-edit lock files to work
around this.

### cspell scans build output

If spellcheck reports hundreds of unknown words from `out/_next` or `.next`, the
ignored build output exists locally. Remove it and rerun spellcheck:

```bash
rm -rf out .next
pnpm run spellcheck
```
