# Recreation Guide for CAR (Claude Artifact Repository)

<!-- cspell:words cmdk exceljs katex papaparse plotly turbopack -->

This guide recreates this repository from scratch with the current toolchain:
Next.js 16, React 19, Tailwind CSS 4, shadcn, pnpm, TypeScript, Prettier, Husky,
and the repo's canonical ESLint policy.

Version facts checked on 2026-06-16:

- `next`: 16.2.9
- `react` / `react-dom`: 19.2.7
- `tailwindcss` / `@tailwindcss/postcss`: 4.3.1
- `zod`: 4.4.3
- `@hookform/resolvers`: 5.4.0
- `typescript`: 6.0.3
- `typescript-eslint`: 8.61.1
- `eslint-formatter-unix`: 9.0.1
- `eslint`: latest is 10.5.0, but this repo uses ESLint 9 because ESLint 10 is not compatible with the current `@typescript-eslint` setup.

References:

- [Next.js installation](https://nextjs.org/docs/app/getting-started/installation)
- [Tailwind CSS with PostCSS](https://tailwindcss.com/docs/installation/using-postcss)
- [shadcn Next.js installation](https://ui.shadcn.com/docs/installation/next)
- [typescript-eslint dependency versions](https://typescript-eslint.io/users/dependency-versions/)

## Prerequisites

- `nvm`
- `git`
- Corepack-managed `pnpm`

## Step 1: Node.js Environment Setup

Install and use the latest LTS version of Node.js:

```bash
nvm install --lts
nvm use --lts
```

Pin the selected Node.js version:

```bash
node -v > .nvmrc
```

Enable Corepack and prepare pnpm:

```bash
corepack enable pnpm
corepack use pnpm@latest
```

## Step 2: Initialize Project via shadcn

Use the current shadcn Next template. This bootstraps Next.js 16, React 19,
TypeScript, Tailwind CSS 4, App Router, and the usual `src`, `app`, `components`,
and `lib` paths.

```bash
pnpm dlx shadcn@latest init -t next
```

When prompted:

- Project name: `car`
- Base color: `zinc`
- CSS variables: yes
- Use `src` directory: yes
- App Router: yes

If shadcn creates a nested `car` directory, move the generated files to the repo
root:

```bash
mv car/* car/.* . 2>/dev/null || true
rmdir car
```

## Step 5: Configure Husky and Git Hooks

Initialize Husky:

```bash
pnpm exec husky init
```

Set the pre-commit hook:

```bash
echo "pnpm exec pretty-quick --staged" > .husky/pre-commit
```

## Step 6: Install All shadcn Components

Install all registry components so `src/components/ui` matches this repo:

```bash
pnpm dlx shadcn@latest add --all -y --overwrite
```

If the current shadcn registry output differs from this repository's canonical
`src/components/ui` files, prefer the repository versions. Do not hand-fix
generated component lint failures during recreation; those failures usually mean
the upstream registry has drifted from this repo's stricter ESLint policy.

## Step 7: Configure Project Scripts

Update `package.json` scripts. Preserve the Husky `prepare` script and include
the unix formatter in the lint script.

```json
{
  "scripts": {
    "build": "next build",
    "dev": "next dev --turbopack",
    "format": "prettier --write --ignore-unknown .",
    "lint": "eslint --cache --fix --format unix .",
    "prepare": "husky; git config blame.ignoreRevsFile .git-blame-ignore-revs",
    "spellcheck": "cspell --config=cspell.json \"**/*.{cjs,js,md,mjs,ts,tsx}\" --no-progress --show-context --show-suggestions",
    "start": "next start",
    "typecheck": "tsc --noEmit"
  }
}
```

## Step 8: Update App Code

`shadcn init` generates `src/app/layout.tsx`, `src/app/globals.css`, and
`src/lib/utils.ts`. Verify `globals.css` starts with the Tailwind CSS 4 imports:

```css
@import "tailwindcss";
@import "tw-animate-css";
```

Replace `src/app/page.tsx` with:

```tsx
"use client";

import dynamic from "next/dynamic";

const Artifact = dynamic(() => import("@/components/artifact"), { ssr: false });

export default function Home() {
  return (
    <main className="min-h-screen antialiased">
      <Artifact />
    </main>
  );
}
```

Create `src/components/artifact.tsx`:

```tsx
function App() {
  return (
    <div className="mx-auto grid max-w-lg gap-4 px-4 py-8">
      <h1 className="text-2xl font-bold">Hello World</h1>
      <p>
        Replace content in{" "}
        <code className="rounded-md bg-secondary p-1 text-sm text-secondary-foreground">
          src/components/artifact.tsx
        </code>{" "}
        with your own content.
      </p>
    </div>
  );
}

export default App;
```

## Step 10: Configure ESLint

Do not use the basic generated ESLint config. After installing the ESLint
dependencies, copy this repo's `eslint.config.mjs` verbatim.

Reason: the config encodes the repo's canonical policy, including Next.js 16 flat
config presets, type-aware TypeScript rules, import-x resolver policy, React
rules, React Hooks rules, promise rules, TanStack Query rules, unused-import
deduping, scoped exceptions, and validation contracts. A shortened config will
not recreate the repo's behavior.

## Step 11: Verify Installation

Run:

```bash
pnpm run format
pnpm run spellcheck
pnpm run typecheck
pnpm run build
```

`pnpm run lint` is useful as a policy check, but it is not a required recreation
gate after refreshing shadcn components. If it reports issues only in generated
`src/components/ui` files, keep the canonical repo component files instead of
hand-editing registry output.

Then start the dev server:

```bash
pnpm run dev
```

Open <http://localhost:3000>. The page should render the "Hello World" artifact
placeholder.

## Common Issues

### ESLint `unix` formatter missing

ESLint 9 no longer ships the unix formatter in core. Install
`eslint-formatter-unix` and keep the lint script as:

```json
"lint": "eslint --cache --fix --format unix ."
```

### ESLint 10 selected by `@latest`

Do not use `eslint@latest` until the `@typescript-eslint` stack supports it for
this repo. Use:

```bash
pnpm add -D eslint@^9
```

### shadcn creates a nested directory

Move files from the nested `car` directory into the repository root:

```bash
mv car/* car/.* . 2>/dev/null || true
rmdir car
```

### pnpm not found

Enable Corepack:

```bash
corepack enable pnpm
```
